구현 기능

◾ ◾ ◾ ◾
1. 프로젝트 개요
🌿 코로나로 인해 온라인 상거래 및 쇼핑의 증가하여 오프라인 지출의 감소 추세
🌿 온라인으로 효율적인 데이터 관리 필요성에 주목
🌿 상품정보, 배송정보를 한번에 관리할 수 있는 프로그램 목적
🌿 랍스터고리 등 정확한 품명을 모르는 상품의 구입이 어려운 문제점을 보완
2. 요약
1.개발환경 : Windows 10, Eclipse
2.사용언어 : Java
3.참여인원 : 6인
4.개발툴   : Eclipse IDE
5.구현기능
▪ 로그인
- 로그인 성공시 안내멘트가 출력
- 로그인 실패시 재입력을 받고 안내멘트가 출력
- 로그인 실패 원인에 대해 각각 다른 안내멘트를 출력
- 관리자 계정과 회원 계정을 구분하고 각각 다른안내멘트를 출력
- 초기화면으로 돌아가기
- 회원정보를 저장하는 파일을 확인

▪ 회원가입
- 회원가입을 통해 고객의 정보를 저장
- 아이디, 비밀번호, 이름, 나이, 전화번호 설정에 길이 및 문자 유효성 검사
- 주소, 베지테리언 단계 유효성 검사
- 전화번호 입력시 '-' 유무 관련없이 데이터 입력 가능
- 중복아이디는 설정 불가
- 제약사항에 위배되는 경우 경고멘트와 함께 재입력
- 사용자를 위한 멈추기 기능
3. 순서도
4. 구현 기능
 1. 로그인
1-1. 로그인 성공
🌿 로그인 화면 입니다.
🌿 올바른 정보를 입력하면 해당 안내멘트가 출력된 후 다음 업무로 진행합니다.
🌿 관리자 계정과 회원 계정을 구분해서 다음 업무에 접근하도록 했습니다.
🌿 id 와 pw에 0을 입력하면 초기화면으로 되돌아갑니다.
1-2. 로그인 실패

🌿 로그인 실패시 출력되는 에러 메시지입니다.
🌿 아이디, 비밀번호, 아이디와 비밀번호 3가지 경우에 대한 유효성 검사를 합니다.
◾ ◾ ◾ ◾
☑ Code
1️⃣ do while을 이용해 조건에 맞지않는 입력을하면 재입력을 받을 수 있습니다.
2️⃣ 유효성 검사를 위해 isaccessibility 메소드를 별도로 생성했습니다.
3️⃣ BufferedReader와 split으로 ','로 구분된 데이터 파일을 읽습니다.
4️⃣ flag 변수를 생성해 if문을 이용해서 사용자 구분과 에러메시지를 출력합니다.
 2. 회원가입
2-1. 회원가입 성공
🌿 회원가입 화면입니다.
🌿 올바른 정보를 입력하면 지정한 파일에 데이터가 저장됩니다.
🌿 모든 정보는 유효성 검사를 합니다.
🌿 마지막에 멈춤기능이 있어 Enter를 눌러 다음화면으로 넘어갑니다.
2-2. 회원가입 실패
🌿 회원가입 실패시 출력메시지입니다.
🌿 문자열 길이, 문자 종류 등 기준에서 벗어나는 정보는 에러메시지를 출력합니다.
🌿 에러메시지 출력 후 재입력을 받고 올바르게ㅇ 입력할때까지 재입력을 받습니다.
◾ ◾ ◾ ◾
☑ Point Code : 전화번호 유효성검사
1️⃣ if(tel.length == 13)은 '-'를 포함한 전화번호가 총 13자리를 의미합니다.
2️⃣ 13자리가 아닌경우는 '-'미기입 또는 잘못기입 2가지 경우로 구분했습니다.
3️⃣ substring()으로 '-'가 없는 번호를 저장할 수 있습니다.
4️⃣ if문과 charAt()로 문자열을 검사합니다.
