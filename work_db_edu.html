4. 구현 기능
 1. 관리자
B-13. 희망 취업처 및 연봉 조회
: 관리자의 학생들의 희망 취업처 및 희망 연봉 조회
1️⃣ 학생들의 희망 취업처 및 희망 연봉 조회
◾ tblwishjob, tblStudent의 inner join으로 조회합니다.
◾ %'||pstudent||'%'으로 학생이름으로 검색할 수 있습니다.
◾ 이름의 일부만 입력해도 관련 데이터가 출력됩니다.
2️⃣ 학생(상담내용 + 희망 취업처 + 희망 연봉) 조회
◾ tblStudent, tblwishjob, tblsugang, tblcounsel의 inner join과 '%'||pstudent||'%'으로 학생관련 데이터를 출력합니다.


B-14. 교육생 팀 편성 조회
1️⃣ 전체 수강생 팀 조회
◾ tblStudent, tblsugang, tblTeam, tbllclass의 inner join과 '%'||pteam||'%'으로 수강생의 팀 데이터를 출력합니다.
2️⃣ 특정 강의의 특정 팀 조회
◾ tblStudent, tblsugang, tblTeam, tbllclass, tblclass inner join과 '%'||pteam||'%', %'||plecture||'%' 으로 출력합니다.

B-15. 교육생 상담조회
◾ tblStudent, tblsugang, tblcounsel, tblmanager의 inner join과 '%'||pname||'%', '%'||ppurpose||'%'으로 학생이름과 목적으로 검색내용을
출력합니다.


 2. 선생님
C-01. 계정관리
◾ 프로시저에 ProTAccountSearch('아이디',비밀번호)를 입력해서 해당 데이터를 출력합니다.
C-02. 강의 스케줄 조회
1️⃣ 교사별 강의 과정 스케줄 조회
◾ distinct, count, subquery, case~end와 tblteacher, tbllsubject, tbllclass, tblsugang, tblclassroom의 inner join으로 출력합니다.

2️⃣ 지정 강의의 과목 스케줄 : 강의명 중복 미허용
◾ case~end, tbllsubject, tbllclass, tblsubject, tblclass, tblbookname, tblclassroom의 inner join으로 데이터를 출력합니다.
◾ 현재 날짜에 따라 강의 진행중 상태가 다르게 나타나도록 기간을 설정했습니다.

3️⃣ 전체 조회
◾ distinct, case~end, tblsugang, tblstudent, tbllclass, tblclass, tblsugangstate, tbllsubject, tblteacher의 inner join으로 데이터를 출력합니다.

4️⃣ 과목번호로 특정과목 조회
◾ distinct, tbllsubject, tbllclass, tblsugang, tblstudent, tblclass, tbllsubject, tblteacher의 inner join으로 데이터를 출력합니다.


C-03. 배점입출력
1️⃣ 배점 입력
◾ if문을 이용해 출석 배점은 20점이상, 총 배점의 합은 100이 되도록 조건을 걸었으며 그 외의 경우는 에러 메시지를 출력합니다.

2️⃣ 배점 수정
◾ 배점 입력과 마찬가지로 같은 조건을 걸었으며 잘못된 입력시 에러 메시지를 출력합니다.

3️⃣ 특정 과목을 조회 : 강의를 마친 과목
◾ 날짜 조건과 inner join으로 해당 데이터를 출력합니다.

4️⃣ 배점번호로 해당 강의 중 종료된 과목 배점 정보 출력
◾ inner join과 where절로 현재 날짜 기준으로 종료된 강의를 출력합니다.

5️⃣ 문제 추가
◾ insert와 if문으로 새로운 문제를 추가 할 수 있습니다.
